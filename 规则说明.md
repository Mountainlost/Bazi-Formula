# 规则说明

## 总体原则
- 先 `calc`，后 `judge`，再 `report`，最后 `audit`
- 所有结论都必须带 `evidence_refs`
- 证据不足时输出保守结果，不得硬判
- `report` 只能使用 `chart.json` 与 `rules.json`
- 解释层不得覆盖规则层
- 当前仓库是规则驱动原型，不是完整商业产品

## 当前已实现
1. 真实四柱排盘
2. 十神基础映射
3. 旺衰评分 v0
4. 候选用神 v0
5. 最终用神受控版 v0
6. 大运起运骨架
7. evidence 审计链

## 当前未实现
1. 完整格局体系
2. 调候
3. 神煞
4. 流年流月
5. 宽输入范围
6. 多流派

## 当前支持范围
1. `calendar_type=solar`
2. `calendar_type=lunar`，但仅做进入既有排盘主链前的受控农历转公历
3. 时区白名单：`Asia/Shanghai`、`Asia/Singapore`、`Asia/Tokyo`
4. `true_solar_time=false`
5. `true_solar_time=true` 仅在 `configs/input_normalization.yaml` 的出生地经度白名单下启用
6. 支持年份范围 `1900-2099`

## 当前规则边界
- 十神只按日主与其他干的五行阴阳关系计算
- 旺衰 v0 只考虑月令、明透扶助、藏干根气和明透克泄耗
- 候选用神 v0 只输出 `candidate`，不等同最终用神
- 最终用神受控版 v0（`final_useful_god_v0`）只有在强弱方向、月令偏置、候选排序和分布约束同时满足时才允许 `determined`
- 如果处于 `balanced`、多候选并列、月令冲突或明显偏多，则必须返回 `insufficient_for_final_determination` 或 `blocked_by_conflict`
- 大运只输出起运骨架，不展开流年流月
- `report` 只能转述规则层，不能新增格局、调候、神煞或越界定论
- 该层不是完整传统命理最终判定系统
