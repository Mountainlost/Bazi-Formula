# 开发流程

## 固定执行顺序
1. `calc`
2. `judge`
3. `report`
4. `audit`

## 各阶段职责
1. `calc`
   - 输入：`用户输入/*.json`
   - 输出：`输出结果/chart.json`
   - 职责：生成真实四柱排盘、排盘依据和大运起运骨架

2. `judge`
   - 输入：`输出结果/chart.json`
   - 输出：`输出结果/rules.json`
   - 职责：生成十神基础映射、旺衰评分 v0、候选用神 v0、最终用神受控版 v0

3. `report`
   - 输入：`输出结果/chart.json`、`输出结果/rules.json`
   - 输出：`输出结果/report.json`
   - 职责：只做结构化转述，不新增超出规则层的判断

4. `audit`
   - 输入：`输出结果/chart.json`、`输出结果/rules.json`、`输出结果/report.json`
   - 输出：`输出结果/audit.json`
   - 职责：检查 schema、evidence、规则一致性、最终用神门槛和报告越界表达

## 当前工程门禁
- Python 3.10 可安装
- Windows PowerShell 可运行
- `pytest` 必须通过
- 黄金样本不得无意漂移
- `report` 不得自由发挥超出 `chart.json` 和 `rules.json`
- 当前仓库保持规则驱动原型定位，不扩展为完整商业产品
- 当前仓库是规则驱动原型，不是完整商业产品

## 最终用神受控版 v0 的流程位置
- 它属于 `judge` 的规则层输出
- 它依赖 `strength`、`provisional_conclusions`、月令偏置和五行分布
- `report` 只能转述它的状态、主元素、次元素、阻断原因
- `audit` 必须检查它是否误把不足证据写成已确定结论

## 当前边界提醒
- 当前支持 `solar`
- 当前支持受控 `lunar -> solar` 输入归一化
- 当前时区仅支持白名单：`Asia/Shanghai`、`Asia/Singapore`、`Asia/Tokyo`
- 当前支持 `true_solar_time=false`
- 当前支持 `true_solar_time=true`，但必须命中出生地经度白名单
- 当前未展开完整格局体系、调候、神煞、流年流月、全球任意时区、缺少闰月标识的闰月输入和多流派支持
